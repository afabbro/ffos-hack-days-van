<X>

Why you should use X-Tags over some other UI component library

  • It's not a component library! (There is a component registry)
  • No dependency on general purpose monolith libs
  • Tinier more modular components than many other libs
  • Going to introduce browser-supported encapsulation methods as implementations arrive and mature
  • Pretty easy to get started with, honestly - let's try one!

Step 1. Register your X-Tags

xtag.register('x-element', {
  onUpgrade: function(){
    // fired once at the time a component 
    // is initially created or parsed
  },
  onInsert: function(){
    // fired each time a component 
    // is inserted into the DOM
  },
  events: {
    'click:delegate(x-element)': function(){
      // activate a clicked x-element
    }
  },
  getters: {
    'children': function(){
      // return all element children
    }
  },
  setters: {
    // Similarly setters would go here
  methods: {
    nextAction: function(){
      // activate next toggler
      // 
    },
    previousAction: function(){
      // activate the previous toggler
    }
  }
});

Step 2. Use your X-Tags in your HTML

  <x-toggle-switch input-name="fnords"></x-toggle-switch>
